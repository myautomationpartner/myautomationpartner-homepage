FILLOUT API RATE LIMIT ANALYSIS
================================================================================

Your Current Usage:
  - n8n metrics: 3 API calls/hour
  - Portal dashboard: 100 clients × 5 users = 500 potential calls

Solution in Option 2:
  - Redis cache (5-minute TTL)
  - With cache: ~120 unique Fillout API calls/day
  - Fillout's limit (estimated): 100-500 calls/minute
  - Your usage: 0.08 calls/second = SAFE

If scaling to 500 clients:
  - No cache: 2,500 calls/day = still fine
  - Estimated limit hit: 1,000+ concurrent portal users

Conclusion: Option 2 handles rate limits gracefully for 100-500 customers.
